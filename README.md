# 项目业务流程

## 📊 系统角色

| 角色 | 权限 | 主要职责 |
|------|------|----------|
| **管理员** | 全部权限 | 系统配置、用户管理、证书管理 |
| **高校用户** | 上传证书 | 录入证书信息、上传PDF文件、上链操作 |
| **验证用户** | 查询验证 | 验证证书真伪、查看证书详情 |

## 🔄 证书生命周期流程

### 1. 证书上传流程

1. **用户登录**
   - 高校用户通过账号密码登录系统
   - 系统验证身份并生成Token

2. **录入证书信息**
   - 填写学生基本信息（姓名、身份证号、院校、专业等）
   - 上传学历证书PDF文件
   - 系统自动计算文件哈希值

3. **阿里云OSS存储**
   - 系统将PDF文件上传至阿里云OSS
   - 生成文件访问URL

4. **区块链上链**
   - 调用智能合约的 `issue` 方法
   - 将证书哈希、学生信息等数据写入区块链
   - 系统等待交易确认并返回交易哈希

5. **本地数据库存储**
   - 保存证书元数据到MySQL数据库
   - 关联区块链交易哈希
   - 生成唯一的证书编号

### 2. 证书验证流程

1. **验证查询**
   - 验证用户访问系统验证页面
   - 输入证书编号或学生信息

2. **数据查询**
   - 系统从数据库查询证书信息
   - 同时从区块链查询对应记录

3. **双重验证**
   - 对比数据库与区块链数据
   - 验证文件哈希是否一致
   - 确认证书状态（有效/已撤销）

4. **结果展示**
   - 显示验证结果
   - 提供证书详情和访问链接
   - 展示区块链交易信息

### 3. 证书管理流程

1. **证书列表**
   - 管理员查看所有证书
   - 支持按状态、时间等筛选

2. **证书详情**
   - 查看证书完整信息
   - 查看区块链交易记录
   - 下载原始PDF文件

3. **证书撤销**
   - 管理员可撤销无效证书
   - 调用智能合约的 `revoke` 方法
   - 更新数据库状态

## 🔗 系统模块交互

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│                 │     │                 │     │                 │

│                 │     │                 │     │                 │
└─────────────────┘     └─────────────────┘     └─────────────────┘
        ↑                        ↑                        ↑
        │                        │                        │
        ↓                        ↓                        ↓
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│                 │     │                 │     │                 │
│  阿里云OSS       │<────│  本地数据库      │<────│  智能合约       │
│                 │     │                 │     │                 │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### 核心交互流程

1. **用户操作** → **前端界面**：用户发起证书上传或验证请求
2. **前端界面** → **后端服务**：提交表单数据和文件
3. **后端服务** → **阿里云OSS**：存储PDF文件
4. **后端服务** → **区块链网络**：调用智能合约方法
5. **区块链网络** → **智能合约**：执行上链操作
6. **后端服务** → **本地数据库**：存储元数据
7. **后端服务** → **前端界面**：返回操作结果
8. **前端界面** → **用户**：展示处理结果

## 📈 数据流向

### 上传流程数据流向
1. **用户输入** → **前端表单**
2. **文件** → **阿里云OSS**
3. **证书信息** → **后端服务**
4. **哈希值** → **区块链**
5. **元数据** → **MySQL数据库**

### 验证流程数据流向
1. **查询条件** → **后端服务**
2. **数据库查询** → **元数据**
3. **区块链查询** → **哈希验证**
4. **OSS访问** → **原始文件**
5. **验证结果** → **前端展示**

## 🛡️ 区块链集成价值

1. **防篡改**：证书哈希存储在区块链，确保数据不可篡改
2. **可追溯**：所有操作都有区块链交易记录
3. **去中心化**：数据分布存储，提高系统可靠性
4. **透明性**：交易记录公开可查，增强信任

## 🔧 技术实现细节

### 智能合约功能
- **issue**：发布新证书到区块链
- **verify**：验证证书真伪
- **revoke**：撤销无效证书

### API接口
- `/api/certificate/upload`：证书上传
- `/api/certificate/verify`：证书验证
- `/api/certificate/list`：证书列表
- `/api/certificate/detail`：证书详情

### 数据安全
- 敏感信息使用环境变量管理
- 密码使用加密存储
- 区块链确保数据不可篡改
- 阿里云OSS提供文件安全存储

## 🛠️ 快速开始

### 环境要求
- Node.js >= 20.19.0
- Java 14
- MySQL 8.0+
- Maven 3.6+
- FISCO BCOS 3.x（区块链环境）
- WeBASE管理平台（可选）

### 1. 前端运行

**安装依赖**
```bash
cd acvs
npm install
```

**配置环境变量**
```bash
# 复制环境变量模板
copy .env.example .env

# 编辑.env文件
//跨域地址 默认为转到后端8081
VITE_API_BASE_URL=http://localhost:8081
//阿里云OSS的认证配置
VITE_OSS_ACCESS_KEY_ID=your_access_key
VITE_OSS_ACCESS_KEY_SECRET=your_secret
```


**启动开发服务器**
```bash
npm run dev
```
访问 http://localhost:5173

### 2. 后端运行

**开启Redis服务**
在Redis文件夹下开启redis-server.exe,如果需要查看Token是否录入 可以在前端开发者页面的控制台看到输出的Jwt-Token
在Redis中通过键值对（GET {tokenString}）tokenString为控制台输出得token 复制即可 ：键等于值得方式存储 并设置了6小时得过期时间

**数据库配置**
```sql
CREATE DATABASE acv_system CHARACTER SET utf8mb4;
```

**修改配置文件**
编辑 `acvsystem/src/main/resources/application.yml`：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/acv_system?useSSL=false
    username: your_username
    password: your_password
```
**配置阿里云OSS**  （如果不使用环境变量）
1.你需要登录学习使用,配置阿里云OSS,有自己的bucket  如果没有可以参考B站教程
2.配置AliOssUtils 工具类
  字段如下 :
      ENDPOINT
      ACCESS_KEY_ID
      ACCESS_KEY_SECRET
      BUCKET_NAME
      region  写在了方法里面 我也不知道自己咋想的
3.请先测试一下能不能正常上传本地文件到自己的buckt     

**初始化区块链** 
所有区块链业务都在 CertificateController.java 中 你需要先拿到合约在fiscobcos上部署然后
拿到合约地址 将其配置到控制类中
需要修改的字段如下：
    //合约地址
    CONTRACTADDRESS = "0xc8ead4b26b2c6ac14c9fd90d9684c9bc2cc40085"

**启动后端服务**
```bash
cd acvsystem
mvn spring-boot:run
```
服务将在 http://localhost:8080 启动

### 3. 区块链环境部署

**方式一：使用FISCO BCOS一键部署**
```bash
# 下载部署脚本
curl -LO https://github.com/FISCO-BCOS/FISCO-BCOS/releases/download/v3.8.0/build_chain.sh
chmod +x build_chain.sh

# 构建4节点联盟链
./build_chain.sh -l "127.0.0.1:4" -p 30300,20200,8545

# 启动节点
bash nodes/127.0.0.1/start_all.sh
```

**方式二：使用Docker（推荐）**
```bash
# 启动FISCO BCOS节点
docker run -d --name fisco-bcos-node \
  -p 20200:20200 -p 8545:8545 \
  fiscoorg/fiscobcos:v3.8.0
```

**部署智能合约**
1. 访问WeBASE管理平台： http://虚拟机IP地址:5002/WeBASE-Front/#/home
2. 上传 `CertHashRegistry.sol` 合约
3. 部署合约并记录合约地址

**配置区块链连接**
修改 `config.toml` 区块链配置：
```
字段
peers=["192.168.1.3:20200", "192.168.1.3:20201", "192.168.1.3:20202", "192.168.1.3:20203"]
端口号不变的情况下 需要修改 ip地址为虚拟机地址 ps：虚拟机我只测试过桥接模式（要与主机在一个网关下）
```



## ⚠️ 项目缺点与限制

### 1. 技术债务
- **区块链依赖复杂**：需要单独部署FISCO BCOS区块链环境
- **配置繁琐**：WeBASE管理平台部署和智能合约配置较为复杂
- **安全性不足**：部分敏感信息硬编码在代码中
- **缺少测试**：缺乏单元测试和集成测试

### 2. 功能限制
- **性能瓶颈**：大文件上传和区块链交易可能存在性能问题
- **用户体验**：部分界面交互不够流畅
- **错误处理**：异常处理机制不够完善
- **文档缺失**：项目文档不够详细

### 3. 部署挑战
- **环境依赖多**：需要配置前端、后端、数据库、区块链等多个环境
- **运维复杂**：区块链节点需要定期维护和监控
- **扩展性有限**：系统架构在高并发场景下可能存在瓶颈

## 🔧 部署建议

### 开发环境
1. **使用Docker Compose**：一键部署完整开发环境
2. **配置本地区块链**：使用FISCO BCOS单节点开发网络
3. **使用WeBASE-Front**：方便管理区块链和部署合约

### 生产环境
1. **多节点区块链**：部署至少4个FISCO BCOS节点
2. **负载均衡**：前端和后端服务使用负载均衡
3. **监控系统**：集成区块链交易监控和应用性能监控
4. **灾备方案**：制定数据备份和恢复策略

### 安全建议
1. **环境变量管理**：使用专业的密钥管理服务
2. **HTTPS配置**：启用全站HTTPS
3. **访问控制**：完善API权限控制
4. **审计日志**：记录所有关键操作日志
